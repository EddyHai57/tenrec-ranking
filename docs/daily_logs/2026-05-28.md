- 发现并修复 DeepFM 初始 FM 二阶项过大问题：PyTorch embedding 默认初始化会让 sum-square trick 输出过宽，改为 FM embedding 小方差初始化，一阶项初始化为 0。
- 诊断 DCN-v2 valid AUC 接近随机的问题：
  - 修复前复跑 `20260528-012130-dcnv2_smoke-dcnv2`，train AUC 从 0.5597 升至 0.6502，valid AUC 从 0.5241 升至 0.5492。
  - 未训练 DCN-v2 在 50k train rows 上 init LogLoss 为 0.7148，平均预测概率为 0.4086，明显偏离 train base rate。
  - 根因是 DCN-v2 默认 embedding 和 cross/output 初始化尺度过大，且 output bias 未按 train base rate 初始化。
  - 修复为小尺度 embedding / cross 初始化，并通过 metadata `pass2.label_counts.train` 计算 `output_bias_init=train_base_rate`。
  - 修复后未训练 init LogLoss 降至 0.5812，接近 batch constant-base LogLoss 0.5796。

## Hist leakage gate check

新增 `scripts/check_hist_leakage.py`，用于验证 `hist_1..hist_10` 在当前 strict user-order split 下是否存在 train hist 直接包含同 user 后续 valid/test target item 的泄漏。

实现边界：

- 纯流式扫描 `data/Tenrec/ctr_data_1M.csv`，按 user block 处理，处理完即释放。
- 复用 `src/tenrec/data.py` 的 `split_counts_for_user()`，不重写 split 逻辑。
- Check 1：计算 train hist set 与 valid/test target set 的 overlap rate。
- Check 2：抽取 1,000,000 个非 padding hist 取值，检查其是否存在于 full file item universe。
- 输出报告写入 ignored 目录：`outputs/inspection/hist_leakage_check.json` 和 `outputs/inspection/hist_leakage_check.md`。
