## 已完成事项

- 扩展 `tests/test_torch_models.py`：
  - DeepFM forward shape；
  - DeepFM 包含一阶线性项和 FM embedding；
  - DeepFM 初始 logit 不被 FM 二阶项撑爆；
  - DCN-v2 forward shape；
  - DCN-v2 包含 cross layers。
- 在 `src/tenrec/models.py` 新增：
  - `DeepFM`；
  - `DcnV2CrossLayer`；
  - `DcnV2`；
  - `build_model()` 注册 `deepfm` 和 `dcnv2`。
- 在 `scripts/train.py` 的 `--model` override 中加入 `deepfm` / `dcnv2`。
- 新增 smoke config：
  - `configs/deepfm_smoke.yaml`
  - `configs/dcnv2_smoke.yaml`
- DeepFM 采用共享 embedding 同时用于 FM 二阶交叉和 DNN 部分；logit 为一阶项、二阶项和 DNN logit 求和。
- DCN-v2 采用 embedding 拼接后，cross network 和 deep network 并行，拼接后输出 logit。
- 发现并修复 DeepFM 初始 FM 二阶项过大问题：PyTorch embedding 默认初始化会让 sum-square trick 输出过宽，改为 FM embedding 小方差初始化，一阶项初始化为 0。
- 诊断 DCN-v2 valid AUC 接近随机的问题：
  - 修复前复跑 `20260528-012130-dcnv2_smoke-dcnv2`，train AUC 从 0.5597 升至 0.6502，valid AUC 从 0.5241 升至 0.5492。
  - 未训练 DCN-v2 在 50k train rows 上 init LogLoss 为 0.7148，平均预测概率为 0.4086，明显偏离 train base rate。
  - 根因是 DCN-v2 默认 embedding 和 cross/output 初始化尺度过大，且 output bias 未按 train base rate 初始化。
  - 修复为小尺度 embedding / cross 初始化，并通过 metadata `pass2.label_counts.train` 计算 `output_bias_init=train_base_rate`。
  - 修复后未训练 init LogLoss 降至 0.5812，接近 batch constant-base LogLoss 0.5796。

## 当前限制

- 两个 run 都使用 `ctr-454e7ccb12f7` 1M user-block sample 的物化数据。
- 两个 run 都使用 `max_train_rows=100000` 和 `max_valid_rows=50000`，属于 head-truncated smoke。
- 当前结果只证明 DeepFM / DCN-v2 已接入训练链路并可运行，不是正式模型指标。
- DCN-v2 smoke 学习较弱，不能解释为 DCN-v2 模型无效；需要 large/full subset 后再判断。
- 修复后 DCN-v2 在 1M head-truncated smoke 中很快出现 train AUC 明显高于 valid AUC，说明存在 tiny sample 过拟合趋势；仍不能作为正式模型结论。
- 当前所有深模型 smoke 都不能用于模型强弱排名：
  - MLP / DCN-v2 在 100k head-truncated train smoke 上出现 valid 早停见顶后 train AUC 继续上升、valid LogLoss 变差的过拟合趋势。
  - DeepFM 在 epoch 2 后 valid LogLoss 变差，也属于同一类 head-truncated smoke 现象。
  - LR / DeepFM / DCN-v2 的 smoke AUC 差异只用于 pipeline / 初始化诊断，禁止解释为正式模型优劣。
- 暂不对 MLP / DeepFM 增加 `output_bias_init: train_base_rate`：
  - MLP 初始 loss 和训练曲线没有暴露类似 DCN-v2 的初始化 bug。
  - DeepFM 已通过小方差 FM 初始化修复初始 logit 过宽问题，overfit 和 train smoke 正常。
  - 现在给 MLP / DeepFM 追加 base-rate bias 属于收敛速度优化，不是 bug fix；GPU 前最后本地动作保持最小变更，避免改变已记录 smoke baseline。
