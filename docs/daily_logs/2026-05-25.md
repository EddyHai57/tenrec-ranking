## Split feasibility 关键统计

split 规则：

- 每个 user 内按文件顺序切分。
- user rows `< 3` 全部 train。
- user rows `>= 3` 按 80/10/10 切分，valid/test 至少各 1 行。
- 不打乱，不去重，保留原始 row order。

基础统计：

| split | rows | users | items | `click=0` | `click=1` | positive rate |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| train | 807,282 | 8,181 | 222,829 | 611,818 | 195,464 | 0.24212605 |
| valid | 96,459 | 8,170 | 54,212 | 70,725 | 25,734 | 0.26678693 |
| test | 96,459 | 8,170 | 55,095 | 75,882 | 20,577 | 0.2133238 |

GAUC 可行性：

- valid：5,677 个有效 GAUC users，覆盖 84,225 行，row coverage rate 0.87316891。
- test：5,021 个有效 GAUC users，覆盖 79,120 行，row coverage rate 0.82024487。

cold item / unseen item：

- valid item 未出现在 train：16,805，rate 0.30998672。
- test item 未出现在 train：16,911，rate 0.30694255。
- test item 未出现在 train + valid：15,784，rate 0.28648698。

feature 使用建议：

- 第一阶段可用：`user_id`、`item_id`、`video_category`、`gender`、`age`。
- 暂时不用：`watching_times`、`hist_1` 到 `hist_10`、`follow`、`like`、`share`。

结论：

- user-block smoke sample 足够支持 dataloader、split 和 GAUC smoke。
- 可以进入最小 `src` / `configs` 设计，但只限 dataloader、split、feature mapping 和 metric smoke。
- 不能报告正式指标。

## 问题 / 风险

- `ctr_data_1M.csv` 约 9.94 GB，不适合用 pandas 直接全量读入本地内存。
- 数据没有显式 timestamp；`Readme.txt` 说明时间信息隐含在用户级 item 顺序中。
- `hist_1` 到 `hist_10` 的构造是否严格只用过去行为仍未验证。
- 用户提供的论文 / 官方材料统计尚未由本地脚本重算，不能写成本项目实验结果。
- `video_category` 存在 `\N`，后续 dataloader 需要明确 missing category 编码。
- 存在重复行、重复 `(user_id,item_id)` 和 click 冲突；去重策略必须先定义数据契约，不能机械删除。
- smoke samples 只是开发样本，不是 full validation / full test。
- user-block split 后 unseen item 比例约 29%-31%，后续 embedding / feature mapping 需要显式 OOV 处理。
- 100k 和 1M sklearn LR 都是 learnability smoke，不是正式 baseline。
- full data 两遍预处理尚未运行，服务器前仍需准备 server runbook 和 full preprocessing 命令。
